# 贡献指南

感谢您对本项目的关注！我们欢迎任何形式的贡献。

## 🤝 如何贡献

### 报告 Bug

如果您发现了 Bug，请：

1. 检查 [Issues](https://github.com/YiXuanHQ/Blog-Template-Astro/issues) 中是否已有相关问题
2. 如果没有，创建一个新的 Issue，包含：
   - Bug 的详细描述
   - 复现步骤
   - 预期行为和实际行为
   - 系统环境信息（操作系统、Node.js 版本等）
   - 相关截图（如有）

### 提出新功能

如果您有好的想法：

1. 先在 [Discussions](https://github.com/YiXuanHQ/Blog-Template-Astro/discussions) 中讨论
2. 获得认可后，创建一个 Feature Request Issue
3. 描述清楚功能的用途和实现思路

### 提交代码

#### 开发流程

1. **Fork 项目**

```bash
# 点击 GitHub 页面右上角的 Fork 按钮
```

2. **克隆到本地**

```bash
git clone https://github.com/YiXuanHQ/Blog-Template-Astro.git
cd your-repo
```

3. **创建分支**

```bash
git checkout -b feature/your-feature-name
# 或
git checkout -b fix/your-bug-fix
```

4. **安装依赖**

```bash
npm install
```

5. **开发和测试**

```bash
# 启动开发服务器
npm run dev

# 构建测试
npm run build

# 预览构建结果
npm run preview
```

6. **提交代码**

```bash
git add .
git commit -m "feat: 添加某某功能"
# 或
git commit -m "fix: 修复某某问题"
```

提交信息格式：

- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式调整（不影响功能）
- `refactor`: 代码重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具相关

7. **推送到 GitHub**

```bash
git push origin feature/your-feature-name
```

8. **创建 Pull Request**

- 访问您 Fork 的仓库页面
- 点击 "New Pull Request"
- 填写 PR 描述，说明改动内容
- 等待审核

#### 代码规范

- 使用 Tab 缩进
- 保持代码简洁易读
- 添加必要的注释
- 遵循项目现有的代码风格

#### Pull Request 规范

- PR 标题清晰明了
- 描述中说明改动的原因和内容
- 如果解决了某个 Issue，在描述中添加 "Closes #issue_number"
- 确保代码能够正常构建
- 一个 PR 只做一件事
